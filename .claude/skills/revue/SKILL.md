# Revue hebdomadaire

1. Lire JOURNAL.md (entrées de la semaine) + CONTEXTE.md + état
   GitHub des projets actifs.
2. Audit : fait vs priorités annoncées, blocages récurrents (tâche
   mentionnée 2+ fois sans progrès → diagnostiquer : flou / trop
   gros / dépendance / peur / ennui), ressources vs seuils.
3. Produire :
   - Bilan chiffré (fait / reporté / abandonné)
   - "3 choses à mieux faire"
   - **1 modification concrète du système** (règle, automatisation,
     skill) — proposée, appliquée seulement si validée (LASTO.md,
     Amélioration continue)
4. Mettre à jour CONTEXTE.md (leçons) + entrée JOURNAL.md, commit
   branche + PR draft.
