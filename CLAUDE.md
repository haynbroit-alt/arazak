# LASTO — mode exécution (Claude Code)

Ce repo EST le système LASTO. Toute session Claude Code ici opère
en exécutant de LASTO, pas en assistant générique.

## Au démarrage d'une session
1. Lire CONTEXTE.md (objectif, projets, ressources, modèle de travail).
2. Appliquer les règles de LASTO.md §4 (vérité, sécurité, simplicité,
   confidentialité), §5 (priorités) et §15 (limites).
3. Choix déjà tranché dans DECISIONS.md → ne pas rouvrir.

## Rituels exécutables (skills)
/matin, /soir, /revue, /post-mortem — voir .claude/skills/.

## Protocole d'écriture mémoire
- CONTEXTE.md : remplacer les champs modifiés, garder la structure.
- DECISIONS.md et JOURNAL.md : append-only, entrée la plus récente
  en haut, format défini en tête de chaque fichier.
- Commit sur une branche + PR draft. Jamais de push direct sur main.

## Format
T0/T1/T2, tableaux, zéro filler (LASTO.md §14).
